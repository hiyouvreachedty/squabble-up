# Squabble-Up — Launch Checklist

Pre-launch QA and go-live checklist for the Squabble-Up website.

---

## Phase 1: Planning & Setup

- [ ] Repo initialized on GitHub
- [ ] Docs folder created (site-structure, brand-notes, content-plan)
- [ ] Domain name secured (squabbleup.ca or squabble-up.com)
- [ ] Vercel project connected to repo
- [ ] Next.js + TypeScript + Tailwind scaffolded
- [ ] Design tokens configured (colors, fonts, spacing)
- [ ] Folder structure matches docs/site-structure.md

---

## Phase 2: Design

- [ ] Logo / wordmark created (SVG)
- [ ] Color palette applied to Tailwind config
- [ ] Typography scale set up
- [ ] Dark mode + light mode theme toggling works
- [ ] Figma mockup or wireframe completed (optional but recommended)
- [ ] Hero image or video sourced
- [ ] Founder photo ready
- [ ] All section content finalized from content-plan.md

---

## Phase 3: Development

### Layout
- [ ] Header built (logo, nav, CTA, mobile drawer)
- [ ] Footer built (nav columns, social, legal)
- [ ] Scroll spy for active nav links
- [ ] Theme toggle working

### Sections
- [ ] HeroSection
- [ ] BrandStatementSection
- [ ] HowItWorksSection
- [ ] ExperienceSection
- [ ] ClassesSection
- [ ] FounderSection
- [ ] TestimonialsSection
- [ ] CTABannerSection

### Functionality
- [ ] Booking CTA links to booking tool or contact form
- [ ] Contact form functional (email or Formspree)
- [ ] Portfolio section links to Tresor Design or case studies
- [ ] All internal links working
- [ ] All external links open in new tab

---

## Phase 4: Content QA

- [ ] All placeholder copy replaced with final copy
- [ ] All [TBD] fields filled in
- [ ] Real testimonials added (or clearly marked as placeholder)
- [ ] Stats (classes held, students trained) are accurate
- [ ] Founder bio finalized
- [ ] Contact email confirmed and working
- [ ] Social links point to correct profiles

---

## Phase 5: Performance & SEO

- [ ] Page title set correctly
- [ ] Meta description added
- [ ] OG image created and linked
- [ ] Favicon added
- [ ] Images optimized (WebP, correct sizes)
- [ ] No broken images or missing alt tags
- [ ] Lighthouse score: Performance > 90
- [ ] Lighthouse score: Accessibility > 90
- [ ] Lighthouse score: SEO > 90
- [ ] Mobile responsive on all common breakpoints
- [ ] Tested on iOS Safari and Chrome Android

---

## Phase 6: Pre-Launch

- [ ] Custom domain connected to Vercel
- [ ] HTTPS / SSL active
- [ ] 404 page created
- [ ] robots.txt in place
- [ ] sitemap.xml generated
- [ ] Google Analytics or Vercel Analytics connected
- [ ] All environment variables set in Vercel
- [ ] Final content review done
- [ ] Tested on desktop, tablet, and mobile

---

## Phase 7: Launch

- [ ] Deploy to production on Vercel
- [ ] Smoke test all sections on live URL
- [ ] Share on Instagram / socials
- [ ] Submit URL to Google Search Console
- [ ] Update GitHub README with live URL badge

---

## Post-Launch

- [ ] Monitor Vercel analytics for first 7 days
- [ ] Collect real testimonials from students
- [ ] Add booking system if needed
- [ ] Add blog or case studies section
- [ ] Set up email capture for class announcements
